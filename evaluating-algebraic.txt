Here is the step-by-step process to run the YACC program for evaluating algebraic expressions in Kali Linux. 🐧💻

1️⃣ Install Required Tools

First make sure flex, bison, and gcc are installed.

sudo apt update
sudo apt install flex bison gcc
2️⃣ Create the YACC File

Create the file eval.y

nano eval.y

Paste the code:

%{
#include <stdio.h>
#include <stdlib.h>

int yylex(void);
int yyerror(const char *s);
%}

%union {
    int val;
}

%token <val> NUM
%type <val> expr input

%left '+' '-'
%left '*' '/'

%%
input:
    expr { printf("Result = %d\n", $1); }
    ;

expr:
      expr '+' expr { $$ = $1 + $3; }
    | expr '-' expr { $$ = $1 - $3; }
    | expr '*' expr { $$ = $1 * $3; }
    | expr '/' expr {
        if ($3 == 0) {
            printf("Division by zero\n");
            exit(0);
        }
        $$ = $1 / $3;
      }
    | '(' expr ')' { $$ = $2; }
    | NUM { $$ = $1; }
    ;
%%

int main() {
    printf("Enter expression:\n");
    yyparse();
    return 0;
}

int yyerror(const char *s) {
    printf("Syntax Error\n");
    return 0;
}

Save:
CTRL + O → Enter → CTRL + X

3️⃣ Create the LEX File

Create the file eval.l

nano eval.l

Paste this code:

%{
#include "eval.tab.h"
#include <stdlib.h>
%}

%%

[0-9]+ { yylval.val = atoi(yytext); return NUM; }
[ \t\n] ;
"+" return '+';
"-" return '-';
"*" return '*';
"/" return '/';
"(" return '(';
")" return ')';
. return yytext[0];

%%

Save and exit.

4️⃣ Generate Parser Code (Bison)
bison -d eval.y

Files created:

eval.tab.c
eval.tab.h
5️⃣ Generate Lexer Code (Flex)
flex eval.l

File created:

lex.yy.c
6️⃣ Compile the Program
gcc eval.tab.c lex.yy.c -o eval -lfl

Executable created:

eval
7️⃣ Run the Program
./eval
8️⃣ Example Inputs and Outputs
Example 1

Input:

Enter expression:
3+5

Output:

Result = 8
Example 2

Input:

Enter expression:
3+5*2

Output:

Result = 13

Because * has higher precedence than +.

⚠️ Important Lab Tips

✔ No spaces needed, but spaces are allowed.
✔ Use the same name for .y and .l files (eval).
✔ Always compile in this order:

bison -d eval.y
flex eval.l
gcc eval.tab.c lex.yy.c -o eval -lfl
./eval
